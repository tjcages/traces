Progress log — readme

← Log

A self-contained progress log + Architecture map for a repo, powered by a reusable Claude Code skill. Tap a section to expand.

Overview

One page, progress/index.html, records every feature and build-out — newest first, each entry a two-second scan. Three tabs:

  • Log — the timeline of entries (status, branch, relative time).
  • Architecture — a living diagram of how the app is wired.
  • Tech stack — plain-English languages, frameworks, libraries & tools.

Statuses: new = "look now" (auto-ages to done after ~2h) · done = landed. Branch badges link to GitHub.

Install into a repo install.sh

One command scaffolds progress/ + scripts/ and wires the rule:

bash
bash ~/.claude/skills/progress-log/scripts/install.sh --mode rule
# --mode hook   → also a Stop hook that blocks finishing if code
#                 changed but the log didn't (the forcing function)
# --mode manual → scaffold only; invoke the skill by hand

Idempotent — never clobbers existing entries; refreshes the engine + scripts.

Use it

Slash commands (or just ask "update the progress log"):

claude
/progress  added the export dialog
/changelog fixed the auth redirect   # alias

The full method agents follow is the scaffolded progress/README.md: one entry per task (edited in place), the status lifecycle, maintain the Architecture tab, one image per task, commit + push.

Live preview preview.json

Default is local-only. Flip the Live preview link toggle at the bottom of the Log (or set progress/preview.json) and the next time an agent works it hosts this page over a Cloudflare tunnel and posts a tappable link.

progress/preview.json
{ "tunnel": true }
What a "skill" is

A skill is a folder of instructions (plus helper files) Claude Code loads. This one lives locally at ~/.claude/skills/progress-log/ — ~ is your home folder, and Claude Code auto-discovers skills there. Nothing is hosted on a server; it's just files.

~/.claude/skills/progress-log/
SKILL.md            # method Claude loads
templates/          # index.html, architecture.html, about.html, …
scripts/            # install.sh, progress-tunnel.sh, progress-server.py
Publish & share npx skills

The registry for npx skills (skills.sh) is just GitHub: any public repo with a SKILL.md at its root is a valid skill. Push this folder to a repo, then anyone installs it with one line:

publish (once)
cd ~/.claude/skills/progress-log
git init && git add -A && git commit -m "progress-log skill"
gh repo create progress-log --public --source=. --push
anyone installs
npx skills add <your-username>/progress-log

Or share the folder directly — zip -r progress-log.zip ~/.claude/skills/progress-log — and they unzip it into their own ~/.claude/skills/.

Controls on the Log
  • Condense — hover the gap between two posts to tuck older ones into a tray.
  • Delete — hover a post; a trash button appears on its right (your view only).
  • Newest first — flips the sort. Live preview link — the tunnel toggle.
